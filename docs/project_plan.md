一  总体目标与实施原则
标准目标：完成一个结构固定、规模可控的小型Transformer INT8定点模型，并在FPGA上实现端到端推理原型。
成果目标：形成可演示的软硬件系统、完整技术文档和测试报告，完成1篇论文投稿或形成可投稿稿件，提交1项发明专利申请。
学习优先：8—9月以数字逻辑、Verilog、PyTorch、量化和FPGA工具入门为主。
范围：不考虑1080P视频生成、自动驾驶、完整大模型等作为硬性任务；摄像头Demo、更低比特量化等仅作为冲刺目标。
软硬件对齐：浮点模型到INT8模型允许小幅精度下降；FPGA结果应与软件INT8黄金模型逐层对齐。
其他说明：项目计划是一个基础参考，并非一个固定DDL。

八  主要风险与处理办法
风险	应对措施
基础不足、学习进度慢	8—9月设置统一入门任务和结对学习；模块先小后大，每项任务都提供参考例程和测试样本。
模型过大、片上资源不足	尽早固定模型规模；资源预算不超过器件容量的70%—80%；必要时减少层数、维度或只实现单Block。
量化后精度下降	先保证INT8，再考虑更低比特；保留浮点和软件INT8两条基线，逐层检查误差来源。
RTL联调拖延	8月先跑最小闭环；接口、位宽和数据格式在10月冻结，未经评审不得频繁修改。
论文、专利最后赶工	从10月起同步积累图表、实验数据和创新点；每次双周汇报检查成果材料进度。
